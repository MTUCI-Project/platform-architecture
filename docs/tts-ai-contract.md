# TTS AI Backend Contract

ONCAST uses Redis Streams and MinIO to decouple `control-plane` from the future
TTS AI backend.

## Responsibilities

`control-plane`:

* receives user requests for text announcements;
* creates an `AnnouncementJob`;
* writes a generation request to Redis Stream `tts.requests`;
* stores the target MinIO location where AI must write the audio file;
* listens to `tts.events`;
* after `ready`, sends the resulting media to `radio-engine` as an announcement.

`tts-ai-backend`:

* reads `tts.requests`;
* generates speech audio;
* writes the audio object to MinIO;
* publishes `ready` or `failed` to `tts.events`.

## Stream: `tts.requests`

Each message is a flat Redis Stream record.

Required fields:

| Field | Meaning |
| --- | --- |
| `jobId` | Stable UUID generated by `control-plane`. |
| `stationId` | Station UUID. |
| `text` | Announcement text. |
| `voiceId` | Requested voice identifier. |
| `language` | BCP-47 language tag, initially `ru-RU`. |
| `bucket` | MinIO bucket where the generated audio must be stored. |
| `objectKey` | MinIO object key reserved for generated audio. |
| `correlationId` | Usually equal to `jobId`; used in logs/events. |

Example:

```text
XADD tts.requests * \
  jobId 2f6d... \
  stationId 7b0d... \
  text "Доброе утро!" \
  voiceId "alena" \
  language "ru-RU" \
  bucket "media" \
  objectKey "stations/7b0d/tts/2f6d.mp3" \
  correlationId "2f6d..."
```

## Stream: `tts.events`

### Ready

Required fields:

| Field | Meaning |
| --- | --- |
| `jobId` | Original job ID. |
| `status` | `ready`. |
| `bucket` | MinIO bucket containing generated audio. |
| `objectKey` | MinIO object key containing generated audio. |
| `versionId` | Optional object version. |

Example:

```text
XADD tts.events * \
  jobId 2f6d... \
  status ready \
  bucket media \
  objectKey stations/7b0d/tts/2f6d.mp3
```

After this event, `control-plane` sends this media to `radio-engine`:

```json
{
  "commandId": "tts:2f6d...",
  "media": {
    "bucket": "media",
    "objectKey": "stations/7b0d/tts/2f6d.mp3"
  },
  "title": "optional title"
}
```

### Failed

Required fields:

| Field | Meaning |
| --- | --- |
| `jobId` | Original job ID. |
| `status` | `failed`. |
| `errorCode` | Machine-readable error. |
| `errorMessage` | Human-readable message. |

`control-plane` marks the job as failed and does not call `radio-engine`.

## Delivery and idempotency

* Redis Streams are treated as at-least-once delivery.
* `jobId` is the idempotency key.
* AI backend may safely republish the same `ready` event.
* `radio-engine` announcement command uses `commandId = "tts:{jobId}"`, so duplicate ready events do not duplicate playback.
* Generated files must be private MinIO objects; public buckets are not used.

## Current MVP defaults

* Language: `ru-RU`.
* Output format: MP3 unless another format is agreed later.
* Playback timing: auto-play immediately after `ready`; the engine mixes the announcement over music using station ducking settings.
