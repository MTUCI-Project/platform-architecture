# Границы сервисов

Система разделена на несколько независимых сервисов.

Каждый сервис отвечает только за свою область и не знает о внутренней реализации остальных компонентов.

## User Backend

Node.js + Express

Отвечает за:

* пользователей;
* JWT;
* станции;
* плейлисты;
* расписания;
* загрузку файлов;
* WebSocket;
* административный API.

User Backend является точкой входа для клиентов и имеет доступ ко всем инфраструктурным компонентам.

---

## Radio Engine

Go

Отвечает за:

* AutoDJ;
* очередь треков;
* планировщик;
* переходы между треками;
* события эфира;
* отправку потока в Icecast.

Движок максимально изолирован от бизнес-логики.

---

## AI Backend

Python

Отвечает только за генерацию TTS.

---

## Infrastructure

* PostgreSQL
* Redis
* MinIO
* Icecast
* Nginx

Каждый сервис должен иметь минимально необходимый набор зависимостей.
