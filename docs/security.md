# Безопасность

Архитектура предполагает минимально необходимый уровень доступа для каждого сервиса.

---

# User Backend

Имеет доступ к:

* PostgreSQL;
* Redis;
* MinIO;
* Radio Engine;
* AI Backend.

User Backend является единственной точкой входа для клиентов.

---

# Radio Engine

Имеет доступ только к:

* MinIO;
* Redis;
* Icecast.

Не имеет доступа к:

* PostgreSQL;
* AI Backend;
* User Backend внутренним данным.

Внешние запросы принимаются только через HTTP API.

---

# AI Backend

Имеет доступ к:

* Redis;
* MinIO.

Не имеет доступа к PostgreSQL.

---

# Icecast

Недоступен из интернета.

Доступен только внутри сети Docker.

---

# MinIO

Недоступен извне.

Доступ осуществляется только через сервисы.

Публичные бакеты не используются.

---

# PostgreSQL

Недоступен извне.

Подключение возможно только из внутренней сети.

---

# Redis

Недоступен извне.

Используется исключительно внутри инфраструктуры.

---

# Nginx

Единственная публичная точка входа.

Отвечает за:

* HTTPS;
* rate limiting;
* reverse proxy;
* скрытие внутренней структуры.

---

# JWT

Используются:

Access Token

TTL:

15 минут

Refresh Token

TTL:

30 дней

Refresh Token хранится в Redis.

Logout приводит к удалению refresh token.

---

# Secrets

Пароли и ключи не хранятся в репозиториях.

Используются:

* .env;
* Docker Secrets;
* Vault (в будущем).

---

# Принцип Zero Trust

Каждый сервис должен иметь только те права, которые необходимы ему для выполнения своей задачи.

Любой сервис может считаться потенциально скомпрометированным.
