# Потоковое вещание

За вещание отвечает Icecast.

Radio Engine отправляет поток в Icecast.

Пользователи никогда не подключаются к Icecast напрямую.

Перед Icecast располагается Nginx.

Схема:

Client

↓

Nginx

↓

Icecast

---

Nginx используется для:

* HTTPS;
* кастомных URL;
* скрытия внутренней инфраструктуры;
* rate limiting;
* кэширования.

Например:

https://radio.example.com/live

вместо

http://icecast:8000/radio

Это позволяет менять внутреннюю реализацию без изменения клиентских приложений.
